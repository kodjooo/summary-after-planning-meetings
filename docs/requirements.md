ТЕХНИЧЕСКОЕ ЗАДАНИЕ
на разработку Telegram-бота для автоматической обработки аудиозаписей встреч
(транскрипция через OpenAI Whisper API + анализ через OpenAI Responses API)

Название системы:
AI Meeting Assistant Bot

Назначение системы:
Разработать Telegram-бота, который принимает аудиозаписи встреч или голосовые сообщения, автоматически выполняет транскрипцию через OpenAI Whisper API, анализирует транскрипцию через OpenAI Responses API (LLM) и возвращает пользователю структурированный протокол встречи.

Бот выполняет функцию AI-секретаря для рабочих встреч и планерок.

------------------------------------------------

1. ЦЕЛЬ СИСТЕМЫ

Создать Telegram-бота, который:

1) принимает аудиозапись встречи
2) автоматически распознает речь
3) анализирует содержание встречи
4) выделяет задачи и исполнителей
5) формирует протокол встречи
6) возвращает результат пользователю в Telegram

Пользователь должен иметь возможность:

• записать встречу на диктофон смартфона
• отправить файл Telegram-боту
• либо записать голосовое сообщение прямо в Telegram
• получить через несколько минут структурированный результат

------------------------------------------------

2. ОСНОВНОЙ PIPELINE СИСТЕМЫ

Общий pipeline обработки:

аудио
↓
Telegram Bot получает файл
↓
скачивание файла с Telegram
↓
конвертация аудио (если необходимо)
↓
OpenAI Whisper API (speech-to-text)
↓
получение транскрипции
↓
OpenAI Responses API (LLM анализ)
↓
формирование протокола встречи
↓
отправка результата пользователю

------------------------------------------------

3. ИСПОЛЬЗУЕМЫЕ API

Система должна использовать OpenAI API.

Используемые модели:

Speech-to-Text:
model = whisper-1

LLM анализ:
через OpenAI Responses API

Пример моделей:

gpt-4.1
или аналогичная LLM модель для анализа текста.

30–60 минут записи.

8. TELEGRAM BOT МОДУЛЬ

Основные функции:

• прием аудио сообщений
• скачивание файлов
• запуск обработки
• отправка результата

Основные команды:

/start
/help

Базовое поведение:

если пользователь отправляет аудио — запускается pipeline обработки.

------------------------------------------------

9. СКАЧИВАНИЕ ФАЙЛОВ TELEGRAM

После получения сообщения бот должен:

1) получить file_id
2) запросить file_path
3) скачать файл

Файл сохраняется во временное хранилище.

------------------------------------------------

10. КОНВЕРТАЦИЯ АУДИО

Для конвертации используется FFmpeg.

Поддерживаемые операции:

ogg → wav
m4a → wav

Пример:

ffmpeg -i input.ogg output.wav

------------------------------------------------

11. МОДУЛЬ ТРАНСКРИПЦИИ

Используется OpenAI Whisper API.

Endpoint:

audio.transcriptions

Параметры:

model = whisper-1
language = ru

На вход:

аудиофайл

На выход:

полная текстовая транскрипция встречи.

------------------------------------------------

12. МОДУЛЬ LLM АНАЛИЗА

После получения транскрипции она отправляется в OpenAI Responses API.

Responses API используется для:

• анализа встречи
• выделения задач
• определения исполнителей
• формирования протокола

------------------------------------------------

13. STRUCTURE REQUEST RESPONSES API

LLM получает:

system_prompt
user_prompt

user_prompt содержит транскрипцию встречи.

Пример:

"Проанализируй транскрипцию встречи:

<транскрипция>

и сформируй структурированный результат."

------------------------------------------------

14. SYSTEM PROMPT

Системный промт задает формат анализа.

Пример:

Ты AI ассистент для анализа рабочих встреч.

На входе дана транскрипция встречи.

Сформируй:

1. краткое резюме встречи
2. основные темы
3. список задач
4. исполнителей задач
5. сроки

Если исполнитель не указан — напиши "исполнитель не назначен".

Если срок не указан — "срок не определен".

Не придумывай информацию, которой нет в тексте.

18. JOB QUEUE (ОЧЕРЕДЬ ЗАДАЧ)

Для стабильной работы необходимо использовать очередь обработки.

Причина:

обработка аудио может занимать несколько минут.

Архитектура:

Telegram Bot
↓
Task Queue
↓
Worker
↓
Whisper API
↓
Responses API

------------------------------------------------

19. ЛОГИКА JOB QUEUE

1) бот получает файл
2) создается задача в очереди
3) пользователь получает сообщение:

"Файл получен. Начинаю обработку."

4) worker выполняет обработку

Этапы задачи:

скачивание файла
конвертация
Whisper транскрипция
LLM анализ

5) результат отправляется пользователю.

------------------------------------------------

20. ОБРАБОТКА ДЛИННЫХ ТРАНСКРИПЦИЙ

Если транскрипция превышает лимит контекста:

1) текст делится на части
2) создаются промежуточные summary
3) выполняется финальный анализ

------------------------------------------------

21. ОБРАБОТКА ОШИБОК

• неподдерживаемый формат
• слишком большой файл
• ошибки API
• пустую запись

Сообщение пользователю:

"Не удалось обработать файл. Попробуйте снова."
